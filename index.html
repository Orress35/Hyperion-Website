What is RS?

RS is a programming language with 5 main branches: RSU, RSL, RSF, RSS and RSM.



Main features

• Tags
• Good Performance
• Beginner-Friendly Syntax
• Access To Low-Level Instructions



Which branch should you choose?

General Use - RSU:

RSU is a fast procedural programming language with C-like syntax. It can be used on all major operating systems (Windows, Linux and MacOS), supports all assembly instructions and most C++ libraries and executes at the same speed as pure C.

Machine Learning and Algorithms - RSL:

RSL is a scripting language based on Python. It supports C-like syntax, variable type declarations and has a few automatic optimizations to make your Python code faster. Sadly, as of right now, RSL has roughly the same performance as pure Python, so I wouldn't recommend using it until the first stable version comes out.

Mathematics and Data Analysis - RSF:

RSF is a functional programming language.

Mobile Development - RSM:

RSM is a object-oriented language based on Java. This branch is currently in early development and I'm planning to fully migrate it from Java to C++ on release, which means that the current description for it will be completely invalid in a few months.



RS for Object-oriented programming

All RS branches support object-oriented programming. However, it's heavily discouraged as it can lead to performance drops, high memory usage and other issues.
